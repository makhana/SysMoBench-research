# SysMoBench Artifacts

Cloned system source trees used for trace harness generation and action-window validation.

## Layout

```
artifacts/
├── <task_name>/                # one subdir per task
│   ├── <upstream_repo>/        # cloned source, instrumented in-place
│   ├── INSTRUMENTATION.md      # how the spec actions map to code (generated by harness-gen skill)
│   └── harness/                # run scripts to regenerate traces
└── README.md
```

## How artifacts get here

Two paths:

1. **Manual clone** (one-time bootstrap per task):
   ```bash
   cd artifacts/
   git clone --depth 1 <url_from_task.yaml> <task_name>
   ```

2. **Harness-generation skill** (agent-driven):
   - Skill: `tla_eval/skills/harness-gen/`
   - Agent reads the task prompt, clones the repo here (if not already present),
     adds trace instrumentation, writes a run.sh, and produces INSTRUMENTATION.md.

## Which systems are ready?

Check `tla_eval/tasks/<task>/task.yaml`'s `tv.repo_path`:
- non-null → artifact exists (locally or referenced)
- null → needs bootstrap via harness-gen skill

## Why not submodules?

Systems evolve; we pin to a specific commit (see `task.yaml`'s `repository.commit`) and
do instrumentation patches that would be painful as submodule overlays. A plain clone
with in-place edits is simpler for a benchmark.

## Disk usage

Each artifact is typically 5-50 MB. All 12 systems should fit in under 1 GB.

## Gitignored

`artifacts/*/` is gitignored — the cloned repos are not committed. Only this README
is tracked.
