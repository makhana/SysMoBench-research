TASK
====

Write a finite-state TLA+ specification of the single-decree Essential Paxos
implementation below. It will be checked with SANY, TLC, captured execution
traces, and separately translated expert invariants.

SCOPE -- SINGLE-DECREE PAXOS ONLY
================================
- Model Proposer, Acceptor, and Learner roles.
- Model Prepare, Promise, Accept, and Accepted messages.
- Acceptor state: promised proposal id, accepted proposal id, accepted value.
- Proposer state: next ballot number, current proposal id, proposed value,
  promises received, and highest previously accepted proposal learned.
- Learner state: each acceptor's latest accepted proposal and the final decided
  proposal/value once a quorum agrees.
- Use simple-majority quorums over Acceptors.
- Model setting a proposer's input value with an auxiliary SetProposal(p, v)
  action. It may appear in Next, but it is not one of the five TV actions.

OUT OF SCOPE
============
- Multi-Paxos, logs, leader election, heartbeats, NACKs, durability, crash
  recovery, retransmission logic, Byzantine faults, fairness, and liveness.

NETWORK MODEL
=============
- Use one global set msgs of in-flight messages.
- Sending adds a message with msgs' = msgs \union {m}.
- Receiving reads a selected message without removing it. This intentionally
  permits loss, duplication, reordering, and repeated delivery.

FINITE BALLOTS
==============
- Declare finite CONSTANTS Acceptors, Proposers, Learners, Values,
  BallotNumbers.
- Define ProposalIDs == BallotNumbers \X Proposers.
- Define explicit lexicographic comparison operators for proposal ids. TLA+
  tuples do not provide a built-in ordering.
- Keep all variable domains finite so TLC can enumerate the initial states.

REQUIRED TV ACTIONS
===================
Define these top-level named actions exactly:
  Prepare(p)
  HandlePrepare(a, m)
  HandlePromise(p, m)
  HandleAccept(a, m)
  HandleAccepted(l, m)

Match the Python handlers closely, including stale and duplicate message
behavior. Define vars, Init, TypeOK, Next, and:
  Spec == Init /\ [][Next]_vars

OUTPUT FORMAT
=============
Output exactly two fenced blocks and no prose:
1. A ```tla block containing module EssentialPaxos.
2. A ```cfg block containing a finite TLC configuration.

The cfg must assign every declared constant, use at least three acceptors, two
proposers, two learners, two values, and three ballot numbers, and include:
  SPECIFICATION Spec
  CHECK_DEADLOCK FALSE

Do not reference Agreement, Validity, Stability, or PromiseMonotonic in this
base cfg. Expert invariants are added separately in Phase 4.

REFERENCE IMPLEMENTATION
========================
{source_code}

Produce the EssentialPaxos specification now.
